# カスタムスケール・コードライブラリ

ユーザーが独自のスケールとコードを追加・編集し、練習モードの選択肢に反映する機能の仕様。  
再生・指板 UI は **練習モード** のみ。編集・CSV・リセットは **ライブラリモード** に集約する。

## 目的

- 練習用 UI を複雑にせず、データ管理を別画面に分離する
- 組み込み定義（マスタ）を壊さず、ユーザー定義だけを永続化する
- CSV でバックアップ・共有・一括取り込みを可能にする

## アプリモード

| モード | 内容 |
|--------|------|
| **練習** | 現行 UI（キー・スケール・コード選択、指板、構成音、再生） |
| **ライブラリ** | スケール/コード/ストロークパターンの一覧・編集、CSV、初期リセット |

ヘッダー右に `[ 練習 | ライブラリ ]` のセグメント切替。既定は **練習**。

## データモデル

### 定義

組み込みと同型:

```typescript
interface ScaleDef {
  id: string;
  name: string;
  tones: readonly string[];
}

interface ChordDef {
  id: string;
  name: string;
  tones: readonly string[];
}

interface StrumPatternDef {
  id: string;
  name: string;
  /** 拍子（例: "4/4", "3/4", "12/8"） */
  timeSignature: string;
  /** 例: "4, 4(>), 4, 4(>)" / "4(>), 8, 8(>)-8, 8, 4"（`-` はタイ、`(>)` はアクセント、`r` は休符） */
  notation: string;
}
```

### 保存先

| キー | 内容 |
|------|------|
| `guitar-practice-settings` | 練習設定（既存）+ `appMode` |
| `guitar-practice-custom-library` | `{ scales: ScaleDef[], chords: ChordDef[], strumPatterns: StrumPatternDef[] }` |

組み込みデータはソースコード（`SCALES` / `CHORDS`）のまま。上書きしない。

### 解決順

`findScaleById` / `findChordById`:

1. 組み込み定義を検索
2. なければカスタム定義を検索

練習モードのプルダウン: **組み込み（先）→ カスタム（後）**。カスタムは表示名に `(カスタム)` を付与。

## ライブラリ UI

### レイアウト

```
[ スケール ] [ コード ] [ ストローク ]     ← サブタブ

┌ 一覧 ──────────┐ ┌ 編集 ─────────────┐
│ ○ Major 組み込み│ │ 名前 [_______]    │
│ ● My Scale      │ │ [▶ 再生] または [▶ 同時][▶ アルペジオ] │
│ + 新規追加      │ │ 構成音 [▼] [削除] │
└────────────────┘ │ + 構成音を追加    │
                   │ [保存] [削除]      │
                   └───────────────────┘

[ CSVダウンロード ] [ CSVアップロード ]
[ 初期状態にリセット ]
```

- **ストロークタブ**: 名前 + 拍子 + パターン（カンマ区切り、`-` はタイ、`(>)` はアクセント、`r` は休符）。組み込み 5 種（4/4×3、3/4、12/8）。
- アクセント `(>)` のストロークは現行 peak と同等、非アクセントは相対的に弱く再生（リピート時のみ）
- **組み込み**: 一覧に「組み込み」バッジ。編集フォームは読み取り専用。「複製してカスタム追加」可。
- **カスタム**: 追加・編集・削除可。
- モバイル: 一覧と編集を縦積み。

### プレビュー再生

- 編集フォームの名前・構成音（未保存の内容を含む）を **C ルート** で試聴できる。
- **スケール**: `▶ 再生`（1 オクターブを順次再生。練習モードと同じ）
- **コード**: `▶ 同時` / `▶ アルペジオ`（練習モードと同じ。9 度以上は 1 オクターブ上）
- 組み込み定義（読み取り専用）でも再生可。

### ID（内部識別子）

- **UI では表示・入力しない**。新規保存時にアプリが自動発行する。
- 形式: `custom-scale-{名前スラッグ}` / `custom-chord-{名前スラッグ}`（英字のみの名前から生成）。日本語のみの名前などスラッグ化できない場合は `custom-scale-s{base36}` 形式。
- 既存カスタム定義の編集では ID を変更しない（名前・構成音のみ更新）。
- CSV の `id` 列はエクスポート・インポート用（上級者向けバックアップ）。画面からの新規追加では不要。

### 構成音入力

- **プルダウン選択**（組み込み定義で使用しているラベル一覧から選択）
- 行の追加・削除（最低 1 行）
- 保存時にバリデーション（後述）

### 初期状態にリセット

- カスタム定義をすべて削除（スケール・コード・ストロークパターン）
- 確認ダイアログ（2 段階推奨: 確認 → 実行）
- 練習設定の `scaleId` / `chordId` が無効になった場合は MVP（C Major / C△7）へフォールバック

### CSV

#### ファイル形式（1 ファイル）

ファイル名の例: `guitar-practice-library.csv`

```csv
type,id,name,tones
scale,my-dorian,My Dorian,"R|2 / 9|m3 / #9|4 / 11|5|6 / 13|b7"
chord,my-m7,My m7,"R|m3 / #9|5|b7"
```

| 列 | 必須 | 説明 |
|----|------|------|
| type | ○ | `scale` または `chord` |
| id | ○ | 英小文字・数字・ハイフン。組み込み ID と重複不可（画面の新規追加では自動発行、CSV 取り込み時は必須） |
| name | ○ | 表示名 |
| tones | ○ | 構成音を `\|` 区切り（カンマを含むラベルに対応） |

- 文字コード: UTF-8
- 1 行目はヘッダー必須
- **ダウンロード**: 組み込み + カスタムのすべて（type 列付き）
- **アップロード**: **カスタム定義のみ置換**（組み込みは変更しない）。CSV に組み込み行が含まれていても **スキップ** する。取り込み前に件数プレビューと確認

#### バリデーション（保存・CSV 共通）

| 項目 | ルール |
|------|--------|
| id | `^[a-z][a-z0-9-]*$`、組み込み ID と重複不可（UI 新規時は自動発行） |
| name | 1 文字以上 |
| tones | 1 つ以上。各ラベルは既知のインターバル表記（`semitoneForToneLabel` で解釈可能） |

## 練習モードとの連携

- ライブラリで削除した ID を練習中に参照している場合、次回描画時に MVP へフォールバック
- 削除したストロークパターン ID を選択中だった場合は組み込み「4分音符」へフォールバック
- 構成音パネル右上: BPM の左に **リズム** プルダウン（ストロークパターン選択）。∞ リピート再生のストローク位置に使用
- 設定 `strumPatternId` は `guitar-practice-settings` に永続化
- ヘッダーのモード切替のみ練習画面に追加（既存コントロールは変更しない）
