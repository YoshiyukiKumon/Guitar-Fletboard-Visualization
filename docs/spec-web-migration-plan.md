## 1. 概要

### 目的

スプレッドシートで運用している「ギター練習ツール」を、**ブラウザ単体で動作する Web アプリ**に移植する。  
キー・スケール・コードに応じて指板上のインターバル（音程）を表示し、練習対象のフレット位置を視覚的に示す。

### スコープ（MVP）

| 含む | 含まない |
|------|----------|
| 指板表示（6 弦 × 24 フレット） | スケール比較（制作中ブロック） |
| キー・スケール・コード選択（master データ準拠） | スケール比較（制作中ブロック） |
| 初期値: **C / Major / △7** | |
| 指板上の **フレットマーカー（● 相当）** | 認証・サーバー保存 |
| 条件付き書式相当の色分け | チューニング変更 |
| localStorage による設定保存 | |
| PC / スマホ向けレスポンシブ UI | |

#### MVP の音楽コンテンツ（確定）

| 項目 | 値 |
|------|-----|
| キー（ルート） | **C** |
| スケール | **C Major** — 構成音: `R`, `2`, `3`, `4`, `5`, `6`, `△7` |
| コード | **C△7** — 構成音: `R`, `3`, `5`, `△7` |

※ 初期表示は C / Major / △7。選択は `localStorage` に保存する。

## 2. 現行スプレッドシートの構造

### 2.1 シート構成

対象は **1 シート**（`gid=1029034944`）に縦積みのブロック構成。

```
┌─────────────────────────────────────┐
│ 指板（ベース・キー C）               │
├─────────────────────────────────────┤
│ スケール（キー選択 + Altered 等）    │
├─────────────────────────────────────┤
│ コードトーン（例: A / m7-5(∅)）      │
├─────────────────────────────────────┤
│ プリセット（C Altered + Am7-5 等）   │
├─────────────────────────────────────┤
│ スケール比較 ← MVP 対象外（制作中）   │
└─────────────────────────────────────┘
```

### 2.2 データの意味

| 要素 | 意味 |
|------|------|
| 列 1〜24 | フレット番号 |
| 行 6 本 | 弦（標準チューニング）。**上＝1 弦・下＝6 弦**（演奏者から見た向き） |
| セル文字 | その位置の音が、選択キーに対するインターバル（例: `R`, `b2 / b9`） |
| 左端ラベル | 開放弦がキーに対して何度か（キー変更で変化） |
| ● 行 | 選択スケール／コードの構成音に一致するフレット位置（Web では指板上のドット模様に置換） |

### 2.3 ロジックの再現方針

グリッドの数値は **キーとチューニングから算出可能** と判断する。  
Web ではセル値の静的コピーではなく、**音楽理論エンジンによる動的生成**を採用する。

## 3. スプレッドシートの数式・条件付き書式について

### 3.1 読取可否

| 項目 | 読取結果 | 補足 |
|------|----------|------|
| 数式（セル参照・関数） | **CSV エクスポートでは取得不可** | エクスポートは計算済みの表示値のみ |
| 条件付き書式（色） | **HTML 公開ビューから一部推定可能** | 背景色とセルクラスの対応は解析済み |
| ● マーカー行 | **表示値として取得可能** | 理論ベースのハイライト計算と照合可能 |

### 3.2 確認した色（参考）

HTML ビューから抽出した背景色の例:

| 色（HEX） | 用途の推定 |
|-----------|------------|
| `#f4cccc` | ルート（R）の強調 |
| `#d9ead3` | スケール構成音 |
| `#6aa84f` | コードトーン／強調構成音 |
| `#d5a6bd` | コード文脈での R 等 |
| `#d0e0e3` | スケール比較ブロック等（MVP 外） |
| `#ffffff` | デフォルト（対象外の音程） |

同一インターバル表記でも **ブロックや選択状態によって色が変わる** ため、スプレッドシートの色を 1:1 で転写するより、**Web 側でルールを定義して再現**する。

### 3.3 理論ベース構築の可否

**可能。推奨する。**

実装するルールの例:

1. 標準チューニング（E-A-D-G-B-E）で各弦・各フレットの音名を求める
2. 選択キー（ルート）から半音差でインターバル ID を決定
3. インターバル ID → 表示ラベル（`b2 / b9` など併記）
4. 選択スケール／コードの構成音集合と一致するセルをハイライト・着色
5. ルート（R）は常に専用色

MVP ではスプレッドシートにない **C Major / C△7** を理論ベースで実装し、**計算結果のゴールデンテスト**で検証する（後述 §7）。

## 4. 要件（確定事項）

### 4.1 表記ルール

| ルール | 内容 |
|--------|------|
| 併記 | `b2 / b9` のように複数表記があるものは **併記を継続** |
| 半音・臨時記号 | スプレッドシートの `＋5`（全角）は Web では **`+5`（半角）** に統一 |
| ルート | `R` |
| 長音 | `△7`（メジャー 7th）など既存表記を維持 |

### 4.2 フレット・表示

| 項目 | 内容 |
|------|------|
| フレット数 | 24（内部データはフル保持） |
| 狭い画面 | **横スクロール**で全フレットを閲覧可能にする |
| レイアウト | PC / スマホ / タブレットそれぞれに最適化（レスポンシブ） |

### 4.3 チューニング

- **レギュラーチューニング（E-A-D-G-B-E）のみ**
- 将来拡張用にドメイン層へ `Tuning` 抽象は残してよいが、MVP UI では変更不可

### 4.4 永続化

- **localStorage** で以下を保存（想定）:
  - 最後に選択したキー
  - スケール / コードの選択
  - 表示オプション（あれば）

### 4.5 UI（スプレッドシートとの差分）

| スプレッドシート | Web アプリ |
|------------------|------------|
| フレット「間」に値が入る表形式 | **フレット線（\|）の直上**にインターバル値を表示 |
| 行＝弦、列＝フレットのセル | 弦ライン + フレット線 + 押さえ位置の背景色 |
| ● 行（スケール用） | **フレット目印インレイ**（3・5・7…）のみ。全マスには置かない |

### 4.6 インターバル表示 — 確定

- 各フレットの値 = **そのフレットを押さえたときの音程**。
- **弦（横線）とフレット（縦線）の交点**にカプセル型ラベルを配置する。
- カプセル: 角丸最大（ピル型）。色はスプレッドシートのセル色に相当:
  - **R**: 赤背景・白文字
  - **スケール音**: 白背景・黒文字
  - **コードトーン**: やや濃い緑背景・黒文字
  - **その他**: 背景なし・半透明白文字のみ（カプセル枠なし・目立たせない）

### 4.7 フレット目印（●）— 確定

実ギターの**装飾インレイ**を表す。スケール／コードの構成音とは別。

| 項目 | 内容 |
|------|------|
| 位置 | **3, 5, 7, 9, 12, 15, 17, 19, 21** フレットのフレット線上、**指板（茶色エリア）の外側・下の専用行**に表示 |
| 12 フレット | **2 つのドット**（二重目印・横並び） |
| 開放弦 | ドットなし |
| 全マス配置 | **しない** |

スケール音・コードトーンは **カプセルの背景色** で表現する（セル全面の色に相当）。

### 4.8 スタンドアロン配布 — 確定

- **Web サーバー不要**。`npm run build` で生成した **単一 HTML** をローカルで開いて動作する。
- 出力: `standalone/guitar-practice.html`（JS / CSS をインライン化）
- 利用: ファイルをダブルクリック、またはブラウザで `file://` として開く

#### 指板 UI イメージ（概念）

```
        3      5      7     12…
        |      |      |      |
        R      2      3      4   ← ラベルは | の上
E ————[====]—|—[====]—|—[====]—|—
        ●      ●      ●      ●   ← インレイ（指板下部・3,5,7…のみ）
```

- フレット番号: 各フレット線の上
- 弦名: 左端固定
- モバイル: フレット領域を横スクロール

## 6. 実装フェーズ

### Phase 1 — MVP（本プロジェクトのゴール）

1. ドメイン: チューニング・キー C・インターバル計算
2. 定義データ: **C Major** スケール、**C△7** コード
3. 指板コンポーネント（フレット上ラベル・弦上ドット・横スクロール）
4. 色分け（セル背景またはラベル背景）+ フレットマーカー
5. localStorage（将来の選択肢用に最小限。MVP は C 固定のためほぼ表示設定のみ）
6. レスポンシブ CSS
7. ゴールデンテスト（C Major / C△7 のマーカー位置・ラベル）

### Phase 2 — 4 ビュー切り替え（実装済み）

スプレッドシートの縦積みブロックに相当する **4 種類の指板ビュー**をタブで切り替える。

| ビュー | 強調 | その他 |
|--------|------|--------|
| 指板ビュー | ルート（R）のみ赤 | ルート以外は白カプセル（スケールビューと同スタイル） |
| スケールビュー | ルート（R）は赤、それ以外のスケール構成音 | 一律「その他」 |
| コードトーンビュー | ルート（R）は赤、それ以外のコード構成音 | 一律「その他」 |
| 複合ビュー | ルート > コード > スケール（MVP 相当） | 上記以外 |

- 選択状態は `localStorage`（`viewMode`）に保存する。
- **ラベル表示**: トグルで **インターバル / 音名** を切り替え（`labelMode` を localStorage に保存）。音名はシャープ表記（例: C, F#, G）。
- **音声再生**: 構成音パネルの「▶ 再生」— スケールは定義順に 1 オクターブを順次再生、コードは全トーンを同時再生（Web Audio API・ファイル不要）。指板カプセルの **タップ／クリック**で弦・フレット位置の実音を単音再生（標準チューニングの MIDI）。
- **音量**: スライダーで 0〜100% 調整（`volume` を localStorage に保存）。
- **キー・スケール・コード選択 UI**（`scaleKeyId` / `scaleId` / `chordKeyId` / `chordId` を localStorage に保存）。**スケールのルートとコードのルートは別指定**。

### Phase 2+（将来）

- スケール比較 UI
- 追加スケール・コード（master 更新時）
- フレット範囲の絞り込み
- PWA / オフライン

Web アプリの **データ定義の正** は `master` シートとする。`scale` シートは UI 参考。

### 8.2 master シートの構成

```
┌──────────────────────────────────────────────┐
│ 指板マトリクス（弦 1〜6 × フレット 0〜24）    │  ← 各セル = 音程クラス 0〜11（半音単位）
├──────────────────────────────────────────────┤
│ 調音（キー名 ↔ 半音 index 0〜11）            │
├──────────────────────────────────────────────┤
│ スケール一覧（名称 + 構成音ラベル列）         │
├──────────────────────────────────────────────┤
│ コード一覧（名称 + 構成音ラベル列）           │
└──────────────────────────────────────────────┘
```

### 8.3 計算根拠（スプレッドシート設計の再現）

実装は次のパイプラインに従う。

```
1. master の指板マトリクスから (弦, フレット) → 音程クラス pc (0〜11) を取得
2. 選択キーの root_pc（調音テーブル）を取得
3. 相対半音: rel = (pc - root_pc + 12) % 12
4. rel → 表示ラベル（例: 2 → "2 / 9", 11 → "△7"）※ §4.1 併記ルール
5. 選択スケール／コードのラベル集合とセルラベルを照合
6. 一致 → 指板上にドット（フレット 1〜24 のみ。開放弦はドットなし）
```

**指板マトリクスの意味**

- キーに依存しない **絶対音程クラス（12 音）** を格納
- 各弦は開放弦の音程クラスから半音ずつ増加（24 フレットで 2 周）
- 例（6 弦 E）: 開放=4, 1F=5, 2F=6, … 11 → 0 → 1 …

**調音テーブル（キー）**

| キー | index |
|------|-------|
| C | 0 |
| C# / Db | 1 |
| D | 2 |
| D# / Eb | 3 |
| E | 4 |
| F | 5 |
| F# / Gb | 6 |
| G | 7 |
| G# / Ab | 8 |
| A | 9 |
| A# / Bb | 10 |
| B | 11 |

### 8.6 MVP と master の対応

| MVP | master での行 |
|-----|----------------|
| C Major | 調音 `C`(0) + スケール `Major` |
| C△7 | 調音 `C`(0) + コード `△7` |

Phase 1 では上記 2 組のみ UI に接続する。§8.4・§8.5 の全件は **JSON / TypeScript のマスターデータ** として `src/domain/data/` に移植する（MVP 後に選択 UI を追加）。
